# Art Value — מערכת ניהול עסקית פנימית

מערכת ניהול פנימית מלאה בעברית (RTL) עבור סטודיו Art Value. דאשבורד, CRM, קליטת לידים, הצעות מחיר ופיננסים — בעיצוב כהה עם הדגשת ליים, מצב כהה/בהיר, ואנימציות.

> **מצב הדגמה** — כל הנתונים נשמרים מקומית בדפדפן (`localStorage`, מפתח `artvalue_data`). אין שרת, אין נתונים אמיתיים, אין סודות.

## הרצה

```bash
npm install
npm run dev
```

פתח את הכתובת שמודפסת (ברירת מחדל: http://localhost:5173).

### בנייה לפרודקשן (סטטי)

```bash
npm run build      # פלט לתיקיית dist/
npm run preview    # תצוגה מקדמית של הבנייה
```

## מודולים

- **דאשבורד** — כרטיסי KPI עם count-up, גרף הכנסות מול הוצאות, דונאט פילוח לקוחות, מגמת הכנסות, פעילות אחרונה.
- **לקוחות (CRM)** — CRUD מלא, חיפוש וסינון, פעולות מהירות (WhatsApp / חיוג), כרטיס מורחב עם הצעות מקושרות.
- **ליד חדש** — טופס רב-שלבי לקליטת ליד. בשליחה הלקוח נכנס כ"ליד".
- **פניות** — רשימת פניות קרות לפי קטגוריה, עם נוסחי הודעה מוכנים, העתקה, וסימון מצב. סימון "פניתי" יוצר אוטומטית ליד ב-CRM (מקור: "פנייה קרה").
- **הצעות מחיר** — בונה הצעה עם פריטים, חישוב מע״מ אוטומטי, מעקב סטטוס, גרסה להדפסה/PDF (בהירה ונקייה), העתקת קישור ושליחה ב-WhatsApp.
- **פיננסים** — רישום הכנסות/הוצאות עם קטגוריות, סיכום חודשי, גרפים, רווח נקי וממוצע לפרויקט.
- **הגדרות** — מצב כהה/בהיר, ייצוא/ייבוא גיבוי, איפוס נתוני הדגמה, הערת אבטחה.

## טכנולוגיה

React 18 · Vite · Framer Motion · Recharts · React Router · Supabase · Heebo (Google Fonts).

## שני מצבי הפעלה (אוטומטי)

האפליקציה בוחרת מצב לפי משתני הסביבה — אותו קוד UI בדיוק בשני המצבים:

| | מצב מקומי (ברירת מחדל) | מצב ענן (Supabase) |
|---|---|---|
| מתי | אין `.env` | קיימים `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY` |
| אחסון | `localStorage` | Postgres בענן |
| התחברות | אין (נכנס ישר) | מסך התחברות (Supabase Auth) |
| סנכרון בין מכשירים | ❌ | ✅ |

### מעבר למצב ענן

1. צור פרויקט ב-[Supabase](https://supabase.com).
2. העתק את `.env.example` ל-`.env` ומלא URL + anon key.
3. הרץ את `supabase/schema.sql` ב-SQL Editor (טבלאות + RLS).
4. צור משתמש ב-Auth → Users → Add user.
5. `npm run build` (Vite צורב את המפתחות בזמן בנייה) ופרוס את `dist/`.
6. התחבר → הגדרות → "ייבוא נתונים מקומיים ל-Supabase".

> RLS מופעל על כל טבלה — כל משתמש ניגש אך ורק לשורות שלו (`user_id = auth.uid()`).
